セッションが切れた後の途中復帰を行う。

## 状態の正（Source of Truth）

**specs/{feature}/tasks.md のステータスカラムが唯一の正**。
復帰時はこのファイルを最優先で確認する。

## Step 1: 現在の状態を把握する

以下の順序で確認する：

1. **tasks.md のステータスを確認する**（最重要）
   - specs/ 配下の全ディレクトリを検索
   - 各 tasks.md の各タスクの status を確認
   - `TODO` / `IN_PROGRESS` / `DONE` / `BLOCKED` を集計
   - `IN_PROGRESS` のタスクが「最後に作業していたタスク」

2. **未コミットの変更を確認する**
   - `git status` で未コミットファイルを確認
   - `git diff` で変更内容を把握
   - 未コミットの変更がある場合はPOに状況を伝える

3. **最新のコミットを確認する**
   - `git log --oneline -10` で直近のコミットを確認
   - 最後のタスクIDを特定

4. **未完了のレビューを確認する**
   - review.md が存在し、REQUEST_CHANGES の場合は対応が必要

## Step 2: 状況をPOに報告する

```
🔄 セッション復帰

前回の状況:
  フィーチャー: {feature-name}
  フェーズ: {Phase N - 名前}
  タスク進捗: {DONE: X / IN_PROGRESS: Y / TODO: Z / BLOCKED: W}
  最後に完了したタスク: {Task-N}
  未コミットの変更: {あり / なし}

残作業:
  - {未完了タスクのリスト}

前回の続きから再開しますか？
```

## Step 3: 再開

POの承認後、以下のルールで再開する：

- **タスク途中で切れた場合（IN_PROGRESS あり）**
  → `git status` で未コミットの変更を確認
  → 変更があればPOに「続行 / 破棄 / 確認」を選択してもらう
  → 続行なら該当タスクの実装を再開

- **フェーズ間で切れた場合（全タスク DONE だが次フェーズ未着手）**
  → 次のフェーズから再開
  → **特に全タスク DONE かつ review.md 未作成の場合は Phase 4（レビュー）を開始する**

- **承認待ちで切れた場合**
  → 成果物を再提示してPOの承認を求める

- **レビューの差し戻し対応中に切れた場合**
  → review.md の未対応指摘を確認し、修正を再開

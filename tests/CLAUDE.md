# Tests ディレクトリルール

> テストは implementer（実装者）が実装と同時に書く。
> 独立したテスターエージェントは存在しない。

## テストルール
- requirements.md の受入基準を100%カバーする
- テスト名は「何を検証しているか」が分かる命名にする
- 正常系・異常系・境界値・エッジケースを含める
- フレーキーテスト（実行ごとに結果が変わるテスト）は禁止

## 構成
- `unit/` — ユニットテスト
- `integration/` — 結合テスト
- `e2e/` — E2Eテスト（必要に応じて）

## テストの書き方
- TDD: テストを先に書く（RED → GREEN → REFACTOR）
- Arrange → Act → Assert パターンに従う
- モックは外部依存にのみ使用する
- テストデータはファクトリ関数で生成する
